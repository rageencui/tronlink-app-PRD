👤 用户故事 & 验收标准
作为 DApp 开发者，我希望 TronLink 能识别并正确处理 HTTP 402 Payment Required 响应体，使我的 AI Agent 应用能通过标准 x402/TRC-402 协议发起链上支付，无需自定义集成逻辑。

作为 TronLink 用户，我希望在 Agent 执行付费操作时，看到结构清晰的支付确认界面，包含金额、收款方、用途标签，并一键授权或拒绝。
  • ✓
    TronLink 能正确解析 TRC-402 格式交易体（amount, token, recipient, deadline, memo）
  • ✓
    DApp 发起 402 请求后，TronLink Extension / Mobile 弹出标准化支付确认弹窗
  • ✓
    SDK 以 npm 包形式发布，提供 TypeScript 类型声明
  • ✓
    交易体解析耗时 <10ms（P99）
  • ✓
    兼容 Web Extension 和 Mobile WebView 两种环境
  • ✓
    不合规或过期的 402 请求给出明确错误提示
🔗 技术依赖 & 风险
TRC-402 协议草案 x402 标准规范 TronLink Extension SDK TronLink Mobile SDK TronWeb 库
  • 高风险：TRC-402 协议标准尚未最终确定，需与 TRON 核心团队保持同步，预留 2 轮协议变更适配 buffer
  • 中风险：Mobile WebView 与 Extension 环境差异，需分别测试签名流程
  • 低风险：402 格式与现有交易格式冲突，通过字段版本号隔离

👤 用户故事 & 验收标准
作为 TronLink 用户，我希望在授权 AI Agent 执行支付时，能精细控制每笔限额、周期总额、白名单、使用场景和有效期，并可随时一键撤销，确保资金安全可控。
  • ✓
    单笔限额（max_per_tx）：超出限额的交易被 Agent 自动拦截，用户收到提醒
  • ✓
    周期总额：支持按日/周/月设置周期上限，周期重置时自动通知
  • ✓
    白名单管理：Agent 只能向白名单地址转账，界面支持添加/删除地址
  • ✓
    场景标签：支持 api_payment / subscription / m2m 三类场景标签，用于审计分类
  • ✓
    有效期（valid_until）：到期自动失效，到期前 24h 提醒用户续期
  • ✓
    一键撤销（revoke）：<3 秒完成链上撤销，撤销后 Agent 立即失效
  • ✓
    审计日志：记录所有授权操作，支持按时间/Agent/场景筛选查询
🔗 技术依赖 & 风险
TRC-20 授权合约 T1: x402 中间件 TronLink 授权界面 后端日志存储 链上事件监听
  • 高风险：合约设计需严格审计，授权字段的链上存储 gas 成本需优化评估
  • 中风险：白名单地址数量无限制可能导致合约存储膨胀，建议设置上限（如 50 地址）
  • 中风险：撤销操作与 Agent 正在进行的交易存在时序竞争，需设计原子锁机制
